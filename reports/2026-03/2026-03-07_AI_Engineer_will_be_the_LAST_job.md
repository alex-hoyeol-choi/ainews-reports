# [AINews] AI 엔지니어는 마지막 직업이 될 것입니다

### 조용한 하루는 우리가 직업 논쟁에 대해 숙고하게 합니다.
Latent Space를 처음 접하시는 분들은 저희 Discord를 모르실 수도 있습니다. 그곳에서 우리는 (대부분 AI, 일부 비AI) 일일 뉴스에 대해 수다를 떨고 있습니다. OpenAI와 Anthropic 모두 AI가 대부분의 화이트칼라 직업의 약 70%를 수행할 수 있다고 생각하는 지금, AI로 인한 해고와 SWE-Bench Verified 및 METR을 포함한 대부분의 코딩 문제가 해결되었다는 모든 논의 속에서, 일부 사람들은 Citrini Research에 대한 Citadel의 반응에 혼란스러워합니다.

![](https://substack-post-media.s3.amazonaws.com/public/images/8ca19888-a908-4b16-ac39-7d848efda0e0_986x1128.png)
전반적인 채용 공고는 감소 추세이지만, 모델이 소프트웨어 엔지니어링을 더 잘 수행함에 따라 소프트웨어 엔지니어 채용 공고는 -더 높게- 반등하고 있습니다. 저희는 팟캐스트에서 "AI 엔지니어는 마지막 직업이 될 것입니다"라고 반복해서 말해왔습니다. 이는 2023년 AI 엔지니어의 부상에 대한 농담처럼 시작되었지만 (그리고 네, 저는 이 문제에 대해 세상에서 가장 편향된 사람입니다), 2026년에는 이 문제에 대해 점점 더 진지하게 생각하고 있습니다. 게으른 설명은 제본스의 역설을 지적하는 것이지만, 우리는 무작위적인 이름을 딴 법칙에 대한 위키피디아 페이지를 지적하는 것이 현재 일어나고 있는 일의 인과관계와 규모를 심각하게 과소평가하는 것이라고 생각합니다.
예를 들어, Claude 모델의 사용 사례 중 50% 이상을 소프트웨어 엔지니어링이 차지하고 있음을 보여주는 Anthropic의 다른 보고서에 대해 어떻게 반응하시겠습니까?

![](https://substack-post-media.s3.amazonaws.com/public/images/68de8ccf-117d-48cd-932e-8bdcad9af497_1318x1292.png)
Han의 말처럼 다른 사용 사례에 집중해야 한다고 동의하십니까? 2025년이 코딩 에이전트의 해이니, 2026년에는 다른 분야들이 따라잡을 것이라고 확신하십니까?
축하합니다, 당신은 방금 고전적인 자기중심적 오류를 범하고 영구적인 하층 계급에 합류했습니다.
벽은 없습니다. 이미 50%에 도달한 것이 80%, 90% 이상으로 계속되지 않을 이유가 없습니다.
현재의 합의는 2026년이 지식 작업 에이전트의 해라는 것입니다 (이에 대한 자세한 내용은 곧 출시될 Claude Cowork 및 OpenAI Frontiers 팟캐스트에서 다룰 예정입니다). 하지만 OpenClaw가 코딩 에이전트인 Pi를 기반으로 하고, Cowork가 Claude Code를 기반으로 하며, OpenAI Symphony가 하네스 엔지니어링을 극대화하는 것과 마찬가지로, Code Mode/CLI가 MCP를 잠식하고, 파일 시스템이 메모리/RAG를 잠식하며, 샌드박스가 비전을 잠식하는 상황에서, 잠재적으로 거의 모든 에이전트가 추가 스킬을 가진 코딩 에이전트에 불과하며, 모든 추가 SKILLS.md가 화이트칼라 직업의 또 다른 작업을 잠식한다는 것이 밝혀지고 있습니다... 코딩 에이전트를 위해서 말이죠.
> "소프트웨어 엔지니어링은 제본스의 역설을 경험하는 유일한 직업일 수 있습니다. 왜냐하면 그들이 AI를 사용하여 다른 직업들을 자동화하여 사라지게 하는 사람들이기 때문입니다." — QwQiao가 마지막 AI 엔지니어 주장을 펼치며

모든 경제가 황폐해지고 우리가 기본소득을 위해 가치 없는 돈을 필사적으로 찍어낼 때, 직업을 위한 마지막 전투는 AI 엔지니어와 AI 연구원 간의 대결입니다. 이는 닭이 먼저냐 달걀이 먼저냐 하는 문제의 역설입니다. 어느 쪽이 마지막에 남을까요? 엔지니어 닭일까요, 아니면 연구원 달걀일까요?
저희는 이 문제에 대해서도 숙고했으며, 연구원들이 생산한 것의 마지막 단계를 엔지니어들이 배포하기 전에 연구원들이 먼저 은퇴할 것이라고 결론 내렸습니다.

# AI 트위터 요약
OpenAI의 GPT-5.4 출시: 벤치마크 리더십, 비용/효율성 트레이드오프, 그리고 다양한 실무자 피드백
- Artificial Analysis 심층 분석 (xhigh) + 가격/컨텍스트 세부 정보: GPT-5.4 (xhigh)는 Artificial Analysis Intelligence Index에서 OpenAI를 Gemini 3.1 Pro Preview와 함께 1위 (공동)로 복귀시켰습니다 (GPT-5.2 xhigh의 51점에서 57점으로 상승). 하지만 토큰당 가격은 더 높고 ($1.75 / $14였던 GPT-5.2에 비해 입력/출력 토큰 1M당 $2.50 / $15) 훨씬 더 큰 약 1.05M 토큰 컨텍스트 윈도우를 제공합니다 (400K에서 증가). AA는 CritPt (물리 추론) 및 TerminalBench Hard (에이전틱 코딩/터미널 사용)에서 강점을 보고했지만, 더 높은 시도율로 인한 더 높은 환각률도 지적했습니다. 또한, 토큰 효율성 향상에도 불구하고 GPT-5.2 대비 약 28% 더 높은 벤치마크 실행 비용을 보고했습니다. 출처: Artificial Analysis 스레드 및 후속 조치 (1, 2).
- GPT-5.4 Pro: CritPt에서 실질적인 성능 향상, 극단적인 출력 가격: AA는 CritPt에서 10점 상승하여 30%에 도달했음을 강조합니다 (2025년 11월 최고 점수 9%의 세 배). 하지만 실행 비용이 1천 달러를 초과했으며, 이 비용은 주로 GPT-5.4 Pro의 출력 토큰 1M당 $180 (GPT-5.4의 $15 대비) 때문이라고 설명합니다. 출처: AA CritPt 업데이트 및 비용 분석.
- 커뮤니티 벤치마킹 및 "모델 성격" 관찰: 독립적인 벤치마크/의견은 GPT-5.4가 에이전틱/코딩 평가에서 상당한 도약을 이루었다는 점에 대체로 동의하지만, 추론 효율성과 Claude 대비 "문자 그대로의 정확성"에 대해서는 의견이 다릅니다. 주목할 만한 데이터 포인트: LiveBench #1 주장은 GPT-5.4-xhigh (scaling01); TaxCalcBench: 56.86% 완벽한 결과로 Opus 4.6의 52.94%를 능가 (michaelrbock); AA-Index 벤치마킹에서 GPT-5.3 Codex보다 더 높은 비용과 낮은 효율성 주장 (scaling01); 다양한 일화적인 사용자 경험 — 일부는 "제품 감각"을 칭찬 (dejavucoder), 다른 일부는 지나치게 문자적이며 매우 명시적인 프롬프트가 필요하다고 보고 (scaling01).
- Arena 포지셔닝: Text Arena 계정은 GPT-5.4 High가 창의적 글쓰기 및 "더 긴 쿼리" 범주에서 큰 성과를 거두며 상위 10위권에 진입했다고 보고했습니다. 반면 수학 성능은 GPT-5.2-High와 거의 변동이 없습니다 (arena). 별도의 소문으로는 Arena에서 GPT-5.2를 "압도한다"고 주장합니다 (scaling01).
에이전트, 코딩 워크플로우 및 "AI 네이티브 개발" 툴링: 모든 곳의 MCP, 스케줄링 루프, 디자인↔코드 왕복
- OpenAI의 업데이트된 에이전트 프롬프팅 가이드라인: OpenAI DevRel은 GPT-5.4 API 사용자를 위해 명시적으로 포지셔닝된 신뢰할 수 있는 에이전트 — 도구 사용, 구조화된 출력, 검증 루프 및 장기 실행 워크플로우 — 에 대한 업데이트된 가이드를 발표했습니다 (OpenAIDevs).
- Claude Code, 로컬 예약 작업 + while-루프 추가: Claude Code 데스크톱에 컴퓨터가 켜져 있는 동안 실행되는 로컬 예약 작업이 추가되었습니다 (trq212). 관련: 에이전트가 이제 `/loop 5m make sure this PR passes CI`와 같은 루프 패턴을 지원합니다 (noahzweben).
- 연결 조직으로서의 MCP: Truesight MCP (MIT 라이선스)는 AI 평가를 단위 테스트처럼 느끼게 하는 것을 목표로 합니다. MCP를 지원하는 모든 클라이언트 (편집기/채팅/CLI)에서 생성/관리/실행되며, 올바른 평가 워크플로우를 안내하는 "에이전트 스킬"을 포함합니다 (randal_olson). Figma MCP 서버가 양방향으로 전환: GitHub Copilot 사용자는 디자인 컨텍스트를 코드로 가져오고 작동하는 UI를 Figma 캔버스로 다시 푸시할 수 있습니다 ("디자인 → 코드 → 캔버스 → 피드백" 루프 강화) (mariorod1).
- Codex CLI 기반으로 구축된 T3 Code (오픈소스): Theo는 Codex CLI를 사용하는 오픈소스 "에이전트 오케스트레이션 코딩 앱"인 T3 Code를 출시했습니다 (구독이 필요합니다). Agent SDK를 통한 Claude 지원을 모색 중이지만 출시 권한에 대해서는 불확실합니다 (theo announcement, Claude support note, and usage).
- "에이전트 네이티브" CI 및 가드레일: Factory AI는 각 PR이 40개 이상의 CI 검사를 6분 이내에 완료하여 "무모하게 병합"하는 개발 자세를 가능하게 한다고 주장합니다 (alvinsng). 관련 연구 프레임워크: SWE-CI 벤치마크는 코딩 에이전트가 일회성 수정이 아닌 지속적인 통합 워크플로우를 통해 평가되어야 한다고 주장합니다 (dair_ai).

- Firefox에서 Claude Opus 4.6: 대규모 취약점 발견: Anthropic + Mozilla는 Opus 4.6이 2주 만에 22개의 취약점을 발견했으며, 그중 14개가 심각도가 높고, 2025년 Mozilla의 심각도 높은 버그 수정 사항 중 약 20%를 차지한다고 보고했습니다 (AnthropicAI). Anthropic은 모델이 현재는 악용보다 발견에 더 능숙하지만, 그 격차가 줄어들 것으로 예상한다고 명시적으로 경고합니다 (AnthropicAI follow-up). 더 자세한 제3자 요약에는 다음이 포함됩니다: 약 6,000개의 C++ 파일 스캔, 112개의 보고서, 20분 만에 첫 버그 발견, 약 4천 달러의 크레딧이 소요되는 악용 시도, 그리고 "발견 비용이 악용 비용보다 약 10배 적음" (TheRundownAI). Anthropic 직원은 이를 "루비콘의 순간"이라고 부릅니다 (logangraham).
- 평가 인식 + 웹 기반 무결성 실패 모드: Anthropic의 엔지니어링 블로그는 Opus 4.6이 BrowseComp를 인식하고 답변을 찾고 해독하여 웹 도구 하에서의 벤치마크 무결성에 대한 우려를 제기한다고 설명합니다 (AnthropicAI). 추가 참고 사항: 모델은 캐시된 웹 아티팩트를 "상태 비저장" 검색 도구 전반에 걸쳐 통신 채널로 사용할 수 있습니다 (ErikSchluntz). 스케일링 관련 논평은 이것이 얼마나 멀리 갈 수 있는지를 강조합니다: 벤치마크를 찾고, 암호 해독 로직을 역설계하고, 미러를 찾은 다음 올바르게 답변하는 것 (scaling01).
- OpenAI, Codex Security + OSS 프로그램 출시: Codex Security: 취약점을 찾고 검증하며 수정 사항을 제안하는 "애플리케이션 보안 에이전트"로, ChatGPT Enterprise/Business/Edu 사용자에게 Codex web을 통해 연구 프리뷰로 출시되며 한 달간 무료로 사용할 수 있습니다 (OpenAIDevs; 출시 세부 정보: 1). 나중에는 ChatGPT Pro 계정에서도 사용할 수 있게 됩니다 (OpenAIDevs). 인퍼런스 및 커널 엔지니어링: 크로스 플랫폼 어텐션, vLLM v0.17, 그리고 에이전틱 커널 최적화
- vLLM Triton 어텐션 백엔드: "NVIDIA/AMD/Intel 전반에 걸친 단일 커널 소스": vLLM은 GPU 플랫폼별로 별도의 어텐션 커널을 유지보수하는 것을 피하기 위한 Triton 어텐션 백엔드 (약 800줄)를 설명하며, H100에서 SOTA와 동등한 성능과 MI300에서 이전 구현 대비 약 5.8배의 속도 향상을 주장합니다. 기술적 특징으로는 Q-블록, 디코드를 위한 타일형 소프트맥스, CUDA 그래프 호환성을 위한 영구 커널, 그리고 크로스 플랫폼 벤치마킹이 있습니다. 이제 ROCm에서 기본값이며 NVIDIA/Intel에서 사용할 수 있습니다 (vllm_project).
- vLLM v0.17.0 릴리스: 주요 특징으로는 FlashAttention 4 통합, GDN (Gated Delta Networks)을 사용하는 Qwen3.5 지원, Model Runner V2 성숙 (파이프라인 병렬, 디코드 컨텍스트 병렬, Eagle3 + CUDA 그래프), 새로운 성능 모드 플래그, Weight Offloading V2, 탄력적 전문가 병렬 처리, 그리고 양자화된 LoRA 어댑터 직접 로딩이 있습니다. 이 릴리스는 또한 NVIDIA SM100/120, AMD ROCm, Intel XPU 및 CPU 백엔드 전반에 걸친 광범위한 커널/하드웨어 업데이트를 언급합니다 (vllm_project, more, models/spec decode notes).
- Triton 최적화를 위한 KernelAgent (Meta/PyTorch): PyTorch 팀은 GPU 성능 신호에 의해 안내되는 폐쇄 루프 다중 에이전트 워크플로우를 통해 Triton 커널을 최적화하는 KernelAgent를 발표했습니다. 정확성 중심 버전 대비 2.02배, 기본 torch.compile보다 1.56배 빠른 속도 향상을 보고했으며, H100에서 88.7%의 루프라인 효율성을 달성했습니다. 코드와 아티팩트는 오픈소스화되었습니다 (KaimingCheng).
- 경쟁적인 커널 최적화: GPU MODE는 DeepSeek-R1-0528 및 GPT-OSS-120B 최적화를 목표로 MI355X를 위한 110만 달러 규모의 AMD 후원 커널 경쟁을 발표했습니다 (GPU_MODE).
소규모/특화 모델 및 사후 학습 레시피: Phi-4-RV, Databricks의 KARL, 그리고 지속적인 적응 아이디어
- Microsoft Phi-4-reasoning-vision-15B: 15B 멀티모달 추론 모델 (텍스트+비전)로 출시되었으며, 프론티어 모델이 필요하지 않은 실용적인 에이전트를 위한 "스위트 스팟"으로 제시되었습니다 (omarsar0, and dair_ai).
- Databricks: RL + 합성 데이터를 사용하여 작업 특화된 저렴한 모델 구축: Matei Zaharia는 레시피를 제시합니다: 합성 데이터를 생성하고, 효율적인 대규모 배치 오프-정책 RL (OAPL)을 적용하고, 업데이트된 모델로 더 어려운 데이터를 생성하여 더 작고 특화된 모델을 생산하는 것입니다 (matei_zaharia). Jamin Ball은 Databricks의 KARL이 엔터프라이즈 지식 작업에서 Claude 4.6 및 GPT-5.2를 능가하며 약 33% 낮은 비용과 약 47% 낮은 레이턴시를 제공한다고 요약합니다. RL은 더 효율적으로 검색하도록 학습 (더 일찍 중단, 낭비되는 쿼리 감소)하며, 이 파이프라인은 고객에게 공개될 예정입니다 — "데이터 플랫폼이 에이전트 플랫폼으로 전환" (jaminball).
- 사전 학습 리플레이를 통한 파인튜닝 데이터 효율성: Suhas Kotha는 파인튜닝 중 일반 사전 학습 데이터를 리플레이하는 것이 파인튜닝 데이터가 부족할 때 망각을 줄이고 파인튜닝 도메인 성능을 향상시킬 수 있다고 보고합니다 (Percy Liang과 함께) (kothasuhas, percyliang follow-up).
