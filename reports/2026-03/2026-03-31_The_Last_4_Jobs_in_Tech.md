# AI 트위터 요약
Claude Code 컴퓨터 사용, Codex Interop, 그리고 코딩 에이전트 하네스 경쟁
- Claude Code, 컴퓨터 사용 기능 확보: Anthropic은 Claude Code 내부에 컴퓨터 사용 기능을 추가하여, Pro/Max 사용자를 대상으로 한 연구 프리뷰에서 에이전트가 앱을 열고, UI를 클릭하며, CLI에서 직접 구축한 것을 테스트할 수 있도록 했습니다. 실질적인 의미는 폐쇄 루프 검증입니다. 즉, 코드 → 실행 → UI 검사 → 수정 → 재테스트로 이어지는 과정인데, 여러 엔지니어들은 이를 신뢰할 수 있는 앱 반복을 위한 빠진 조각이라고 불렀습니다. 특히 개방형 데스크톱 에이전트와 비교할 때 더욱 그렇습니다 (Claude announcement, @Yuchenj_UW on the “eyes” unlock, @omarsar0).
- 교차 에이전트 구성이 표준이 되고 있습니다: OpenAI는 Claude Code용 Codex 플러그인을 출시했습니다. 이 플러그인은 맞춤형 글루 코드 대신 ChatGPT 구독을 사용하여 Anthropic의 툴체인 내에서 리뷰, 적대적 리뷰 및 "구조" 플로우를 트리거할 수 있습니다. 이는 플러그인의 참신함이라기보다는 코딩 스택이 모놀리식 제품이 아닌 구성 가능한 하네스가 되고 있다는 신호라는 점에서 주목할 만합니다 (plugin by @dkundel, usage thread by @reach_vb, open-source note). 별도로 OpenAI는 심야 Codex 작업이 더 오래 실행되며, 오후 11시경에 시작된 작업이 3시간 이상 실행될 가능성이 60% 더 높다고 밝혔습니다. 이는 리팩터링 및 계획을 백그라운드 에이전트에 위임하는 새로운 패턴과 일치합니다 (OpenAI Devs).
- 하네스 품질이 이제 명백히 1차 변수가 되었습니다: Theo는 Opus가 Claude Code보다 Cursor에서 약 20% 더 높은 점수를 기록했으며, 더 나아가 클로즈드 소스 하네스는 커뮤니티가 회귀를 진단하거나 수정하기 어렵게 만든다고 주장했습니다 (performance gap claim, closed-source critique). 이러한 주제는 피드 전반에 걸쳐 반복되었습니다. 모델 기능 격차는 좁아지고 있지만, 툴링, 프롬프트/런타임 오케스트레이션 및 리뷰 루프는 여전히 큰 실제적 차이를 만들어내고 있습니다.
Hermes Agent의 빠른 부상, 다중 에이전트 프로필, 그리고 오픈 하네스 생태계
- Hermes는 이번 주 돌풍을 일으킨 오픈 에이전트 스택이 되었습니다: Nous는 주요 Hermes Agent 업데이트를 출시하여 OpenClaw/OpenClaw와 유사한 설정에서 대규모 마이그레이션을 이끌었습니다. 사용자들은 더 나은 압축, 불필요한 요소 감소, 더 강력한 적응성 및 더 빠른 출시 주기를 강조했습니다 (Nous release, Teknium’s multi-agent profiles, community migration examples, another). 새로운 다중 에이전트 프로필은 각 봇에 자체 메모리, 스킬, 기록 및 게이트웨이 연결을 제공하여 Hermes를 "개인 비서"에서 재사용 가능한 에이전트 OS 추상화로 전환시키고 있습니다.
- 트레이스, 원격 제어 및 자기 개선을 중심으로 생태계가 형성되고 있습니다: 여러 프로젝트가 Hermes를 핵심 인퍼런스 이상으로 확장하고 있습니다. @jayfarei의 opentraces.ai는 에이전트 트레이스를 정제하고 Hugging Face에 게시하여 분석, 평가, SFT 및 RL에 활용할 수 있는 CLI/스키마/리뷰 플로우를 제공합니다. @kaiostephens는 약 4,000개의 GLM-5 Hermes 트레이스를 HF에 업로드했습니다. @IcarusHermes는 에이전트가 자신의 결정을 기록하고, 데이터를 내보내고, 자신의 기록을 기반으로 더 작은 후속 모델을 파인튜닝하며, 더 저렴한 모델로 전환하는 통합을 설명했습니다. @winglian의 ARC는 E2E 암호화를 통해 원격 브라우저 기반 모니터링/제어 기능을 추가합니다.
- 오픈 소스 vs 독점 에이전트 인프라가 활발히 논의되고 있습니다: @ClementDelangue는 개인 정보 보호와 내구성 모두를 위해 오픈소스 에이전트 도구가 기본적으로 오픈소스 모델을 사용해야 한다고 명시적으로 주장했습니다. 이와 동시에 벤더들은 알려진 문제점을 해결하고 있습니다. @fchollet은 샌드박싱, 승인, RBAC 및 감사 추적 기능을 갖춘 더 안전한 OpenClaw 스타일 어시스턴트인 PokeeClaw를 강조했습니다. Z AI는 API 키가 필요 없고 GLM-5-Turbo를 선택적으로 사용할 수 있는 로컬 OpenClaw 런타임인 AutoClaw를 출시했습니다.
Qwen3.5-Omni, GLM-5-Turbo/AutoClaw, 그리고 로컬/에이전틱 특화로의 추진
- Qwen3.5-Omni는 주요 멀티모달 릴리스입니다: Alibaba는 네이티브 텍스트/이미지/오디오/비디오 이해, 스크립트 수준 캡셔닝, 내장 웹 검색 및 함수 호출 기능을 갖춘 Qwen3.5-Omni를 발표했습니다. 특히 모델이 음성 시각 지시를 통해 웹사이트/게임을 구축하는 뛰어난 "오디오-비주얼 바이브 코딩" 데모가 돋보였습니다. 보고된 기능으로는 10시간 오디오 / 400초의 720p 비디오, 113개 음성 인식 언어, 36개 음성 언어 지원이 포함됩니다. Alibaba는 오디오에서 Gemini 3.1 Pro를 능가하며 일부 설정에서는 AV 이해에서 Gemini 3.1 Pro와 동등하다고 주장합니다 (launch thread, demo thread, additional demo). @kimmonismus의 유용한 주의사항: 여기서 "omni"는 임의의 멀티모달 생성이 아닌 멀티모달 입력을 해석하는 것에 관한 것입니다.
- Z AI는 에이전틱 워크로드에 맞춰 계속 튜닝하고 있습니다: Artificial Analysis는 Z AI의 독점적인 에이전트 최적화 변형인 GLM-5-Turbo를 평가했습니다. 이 모델은 AA Intelligence Index에서 47점을 기록하여 오픈 웨이트 GLM-5 (추론)의 50점보다 약간 뒤처졌지만, GDPval-AA에서는 GLM-5의 1408점보다 앞선 1503점을 기록했습니다. 이는 모델이 광범위한 벤치마크 최대주의보다는 실제 에이전트 워크플로우에 맞춰 튜닝되었다는 주장을 뒷받침합니다.
- 특화된 오픈 모델이 점점 더 배포 패턴이 되고 있습니다: 여러 트윗이 동일한 논지로 수렴했습니다. 기업들은 범용 API를 무기한으로 임대하기보다는 독점 데이터에 대해 오픈 모델을 소유하고 특화하는 경향이 점점 더 커질 것이라는 것입니다 (@oneill_c, @ClementDelangue). 이를 뒷받침하는 증거로는 Claude 4.6 Opus에서 디스틸레이션되어 몇 주 동안 HF에서 트렌드였으며 4비트에서 16GB에 적합하다고 알려진 Qwen3.5-27B 모델(Unsloth, @Hesamation)부터 llama.cpp 및 MLX와 같은 로컬 런타임에 대한 증가하는 열기까지 다양했습니다.
로컬 인퍼런스 및 시스템: 10만 개의 llama.cpp, MacBook의 Flash-MoE, 그리고 웹/서빙 툴체인
- 로컬 AI는 llama.cpp가 GitHub 별 10만 개를 달성하며 상징적인 이정표를 세웠습니다: @ggerganov의 성찰은 2026년을 로컬 에이전틱 워크플로우의 잠재적인 돌파구 해로 규정하며, 유용한 자동화는 프론티어 규모의 호스팅 모델을 필요로 하지 않으며 올바른 휴대용 런타임 스택이 절대적 규모보다 더 중요하다고 주장했습니다. 이 게시물은 또한 교차 하드웨어, 벤더 종속 없는 인프라의 중요성을 강조했습니다.
- Apple Silicon의 Flash-MoE가 큰 주목을 받았습니다: 널리 공유된 게시물은 Qwen3.5-397B가 SSD에서 가중치를 스트리밍하고 활성 전문가만 로드하는 순수 C + Metal 엔진을 사용하여 48GB MacBook Pro에서 4.4 토큰/초로 실행될 수 있으며, 인퍼런스 중 약 5.5GB RAM을 사용한다고 주장했습니다 (summary thread). 관련 작업으로는 MLX 위에 MoE 경로만 최적화하는 anemll-flash-mlx와 AI Toolkit의 새로운 Apple Silicon 지원이 있습니다.
- 웹 및 서빙 스택도 발전했습니다: Transformers.js v4는 브라우저/Node/Bun/Deno 전반에 걸쳐 WebGPU 백엔드를 추가하여 주요 성능 향상과 200개 이상의 아키텍처를 지원합니다. vLLM-Omni v0.18.0은 324개의 커밋, 프로덕션 TTS/옴니 서빙, 통합 양자화, 확산 런타임 리팩터링 및 십여 개 이상의 새로운 모델을 출시했습니다. 음성 측면에서는 Artificial Analysis가 Cohere Transcribe를 다루었습니다. 이는 2B 컨포머 인코더-디코더로, Apache 2.0 라이선스이며 14개 언어로 학습되었고, 4.7% AA-WER을 달성하며 약 60배의 실시간 전사 속도를 보였습니다.
에이전트 연구: 자연어 하네스, Meta-Harness, 비동기 SWE 에이전트, 그리고 파일 시스템을 통한 긴 컨텍스트
- 하네스 엔지니어링이 자체적인 연구 분야가 되고 있습니다: 칭화/선전 대학의 자연어 에이전트 하네스에 관한 논문은 LLM이 하드코딩된 하네스 규칙 대신 SOP에서 오케스트레이션 로직을 실행하도록 제안했습니다. 이는 컨텍스트 예산이 증가함에 따라 여러 실무자들이 매우 복잡하지만 그럴듯하다고 생각하는 방향입니다 (@rronak_ summary). Meta는 Meta-Harness를 통해 이 아이디어를 더욱 발전시켰습니다. 이는 기본 모델뿐만 아니라 코드, 트레이스 및 점수에 걸쳐 하네스를 엔드투엔드로 최적화하는 방법입니다. Haiku 에이전트 중 TerminalBench-2에서 1위를 차지했으며 텍스트 분류 및 전이에서 강력한 성능 향상을 보였습니다 (@yoonholeee, explainer by @LiorOnAI).
- 비동기/다중 에이전트 SWE 설계가 더 강력한 실증적 뒷받침을 얻었습니다: CMU의 CAID 논문은 관리자 에이전트, 의존성 그래프, 격리된 Git 워크트리, 자체 검증 및 병합을 사용하는 중앙 집중식 비동기 격리 위임을 주장합니다. 보고된 이득은 PaperBench에서 단일 에이전트 기준선 대비 절대값으로 +26.7, Commit0에서 +14.3이었으며, 이는 동시성과 격리가 단순히 하나의 에이전트에 더 많은 반복을 주는 것보다 낫다는 것을 시사합니다 (@omarsar0 summary).
- 코딩 에이전트를 긴 컨텍스트 프로세서로 재구성하는 것은 가장 흥미로운 재구성 중 하나입니다: @dair_ai가 강조한 논문은 방대한 코퍼스를 디렉토리 트리로 취급하고, 텍스트를 컨텍스트 윈도우에 채워 넣거나 순전히 리트리벌에 의존하는 대신, 상용 코딩 에이전트가 셸 명령과 Python으로 이를 탐색하도록 합니다. 보고된 결과는 BrowseComp-Plus (7억 5천만 토큰)에서 88.5%로 이전 최고 기록 80%를 능가했으며, 3조 토큰까지 작동합니다.
학습, 최적화, 평가, 그리고 프로덕션 사례 연구
- Muon이 의미 있는 시스템/수학적 최적화를 얻었습니다: Gram Newton-Schulz는 Muon의 Newton-Schulz 단계를 드롭인 대체할 수 있는 것으로, 큰 직사각형 행렬 대신 더 작은 대칭 XXᵀ Gram 행렬에서 작동하여 Muon을 최대 2배 빠르게 만들면서 검증 퍼플렉시티를 0.01 이내로 유지한다고 보고되었습니다. 이 작업은 @tri_dao로부터 실제로 중요한 학제 간 선형 대수 + 고속 커널 결과라는 칭찬을 받았습니다.
- 두 가지 실제 구현 세부 사항이 돋보였습니다: Ross Wightman은 LLM 학습 코드에서 미묘하지만 중요한 PyTorch trunc_normal_ 오용 패턴을 지적했습니다. 기본 a/b는 표준 편차가 아닌 절대값이므로 많은 코드베이스가 사실상 전혀 잘라내지 않고 있다는 것입니다. 그는 또한 나이틀리 빌드에서 나중에 수정된 수치적 이상 현상도 언급했습니다. 애플리케이션 계층에서는 Shopify의 DSPy 사례 연구가 경제성 측면에서 주목할 만했습니다. 한 슬라이드는 비즈니스 로직을 분해하고, DSPy로 의도를 모델링하며, 성능을 유지하면서 더 작고 최적화된 모델로 전환함으로써 연간 550만 달러에서 7만 3천 달러로 비용이 절감되었음을 강조했습니다 (follow-up).
- 새로운 평가/벤치마크는 계속해서 격차를 드러냈습니다: World Reasoning Arena는 가설/세계 모델 추론을 목표로 하며 인간과의 상당한 격차를 보고합니다. Tau Bench의 새로운 뱅킹 도메인은 현실적인 698개 문서 지원 환경을 추가했는데, 여기서 최고 모델도 여전히 약 25%의 작업만 해결합니다. 한편, @Zulfikar_Ramzan이 강조한 스탠포드 주도 논문은 아첨하는 AI가 사용자의 확신을 높이는 반면 관계를 회복하려는 의지를 감소시킬 수 있으며, "유용성" 지표가 사회적으로 해로운 행동을 가릴 수 있음을 강조했습니다.
참여도 높은 인기 트윗
- Claude Code 컴퓨터 사용: Anthropic의 출시는 이번 목록에서 가장 큰 기술 제품 출시였으며, 일상적인 코딩 에이전트 UX에 가장 중요한 영향을 미칠 것으로 보입니다 (announcement).
