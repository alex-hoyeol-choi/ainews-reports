# [AINews] Pull Request의 종말 (2005-2026)

### 조용한 날, Pull Request의 종말을 보도합니다

![Latent.Space's avatar](https://substack-post-media.s3.amazonaws.com/public/images/db0f8d45-1eb8-4c02-a120-650d377ee52d_640x640.jpeg)
Latent.SpaceShare코드 리뷰의 종말에 이어, Pull Request도 다음 차례가 될 수 있습니다.
지난 15년간 코딩을 배운 사람이라면 Git, GitHub, Pull Request 없는 삶을 상상하기 어렵겠지만, 그 이전의 시대가 있었고, 그 이후의 삶이 올 수도 있습니다.
Pull Request는 2005년에 논쟁의 여지는 있지만 발명되었고, GitHub에 의해 성공적으로 대중화되었으며, 불과 21년 후 GitHub는 역사상 처음으로 사람들이 오픈소스 레포에서 Pull Request를 비활성화할 수 있도록 허용하고 있습니다 (이전에는 이슈만 비활성화할 수 있었습니다).

![](https://substack-post-media.s3.amazonaws.com/public/images/d974198b-3217-4de1-ae09-e8aba5710e67_1364x708.png)
코드 분야의 생성형 AI 부상은 한동안 Pull Request의 임박한 종말을 예고했습니다. Pete Steinberger는 (Theo와 함께) Pull Request 대신 Prompt Request를 선호하는 것으로 잘 알려져 있습니다 (여러 가지 이유로, 예를 들어 1) 병합 충돌 없음, 2) 관리자가 코드를 살펴보는 것보다 프롬프트를 수정/추가하기 더 쉬움, 3) 무해해 보이는 PR에 악성/안전하지 않은 코드가 몰래 들어갈 가능성이 적음). Mitchell Hashimoto와 Amp Code와 같은 다른 사람들은 신뢰할 수 없는 코드 기여를 처리하기 위한 "평판" 기반 시스템을 만들었습니다.
Building for Trillions of Agents에서 Aaron Levie는 "나아갈 길은 에이전트가 원하는 소프트웨어를 만드는 것"이라고 언급했습니다. 인간은 인간의 협업을 위해 Git을 발명했습니다. 코드 흐름에서 인간 병목 현상을 제거하면 Git 기반 워크플로우가 적합하지 않을 수 있다는 점이 점점 더 분명해지고 있습니다.
그리고 코드 리뷰가 사라지고, Pull Request도 사라진다면... Git 자체의 종말까지 얼마나 걸릴까요?
> 2026년 4월 14일~4월 15일 AI 뉴스입니다. 12개 서브레딧, 544개 트위터 계정을 확인했으며, 추가 디스코드 채널은 확인하지 않았습니다. AINews 웹사이트에서 지난 모든 이슈를 검색할 수 있습니다. 참고로, AINews는 이제 Latent Space의 한 섹션입니다. 이메일 수신 빈도를 선택/해제할 수 있습니다!

- 출시 직후 주목할 만한 생태계가 형성되었습니다: @CloudflareDev, @modal, @daytonaio, @e2b, @vercel_dev 모두 공식 샌드박스 통합을 발표했습니다. 실용적인 패턴은 스테이트리스 오케스트레이션 + 상태 저장 격리 워크스페이스로 수렴되고 있습니다. @akshat_b의 GPU 샌드박스, 서브에이전트, 영구 메모리, 포크/재개 스냅샷을 갖춘 Modal 기반 ML 연구 에이전트와, @whoiskatrin의 샌드박스에서 작업을 실행하고 출력을 로컬로 복사하는 Python 에이전트를 위한 Cloudflare 가이드 등 예시 빌드가 이미 등장했습니다.
