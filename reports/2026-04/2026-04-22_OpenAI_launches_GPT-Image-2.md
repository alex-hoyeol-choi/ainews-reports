### Cursor가 xAI와 100억 달러 계약을 체결하고 600억 달러에 인수할 권리를 얻었습니다.
오늘 Cursor의 xAI와의 600억 달러 규모 거래가 거의 헤드라인을 장식할 뻔했지만, 순전히 재정적인 이야기(동기에 대한 그럴듯한 분석은 여기에서 확인하실 수 있습니다)이므로, 오늘 OpenAI의 GPT-Image-2 대규모 출시에 주요 기사를 할애합니다.

# AI 트위터 요약
OpenAI의 GPT-Image-2 출시와 진지한 제품 표면으로서의 이미지 생성의 귀환
- GPT-Image-2는 오늘 가장 명확한 제품 출시입니다: OpenAI는 ChatGPT Images 2.0과 기반 gpt-image-2 모델을 ChatGPT, Codex, 그리고 API 전반에 걸쳐 출시했으며, 더 강력한 텍스트 렌더링, 레이아웃 충실도, 편집, 다국어 지원, 그리고 이미지에 대한 "사고(thinking)"를 강조했습니다. OpenAI는 이 모델이 사고(thinking) 모델과 결합될 때 웹을 검색하고, 여러 후보를 생성하며, 자체적으로 출력을 확인하고, 슬라이드, 인포그래픽, 다이어그램, UI 목업, QR 코드와 같은 아티팩트를 생성할 수 있다고 말합니다 (출시 스레드, 사고/이미지 기능, 가용성, API 게시물). 이 모델은 이미 Figma, Canva, Firefly, fal, Hermes Agent를 포함한 다운스트림 도구에 통합되고 있습니다.
- 벤치마크는 특히 실용적인 이미지 작업에서 큰 도약을 시사합니다: Arena는 GPT-Image-2가 모든 Image Arena 리더보드에서 1위를 차지했다고 보고하며, text-to-image에서 1512점, single-image edit에서 1513점, multi-image edit에서 1464점을 기록했고, 다음 모델보다 text-to-image에서 242 Elo라는 놀라운 선두를 보였습니다 (Arena 요약, 카테고리 분석, 트렌드 차트). 독립적인 반응들은 동일한 주제로 수렴했습니다: 이는 단순히 더 예쁜 예술이 아니라, UI, 목업, 문서화, 생산성 시각 자료, 그리고 레퍼런스 기반 디자인 루프를 위한 더 유용한 모델이라는 것입니다 (@gdb, @nickaturley, @mark_k, @petergostev). 가장 흥미로운 시스템적 함의는 이미지 생성이 코딩 에이전트의 프론트엔드가 되고 있다는 점입니다: UI 사양을 이미지로 생성한 다음, Codex 또는 다른 코드 에이전트가 해당 시각적 레퍼런스를 기반으로 구현하도록 하는 것입니다.
에이전트 인프라: Hugging Face의 ml-intern, Hermes 확장, 그리고 연구/런타임 하네스의 부상
- Hugging Face의 ml-intern은 이 세트에서 가장 강력한 오픈 에이전트-인-더-루프(agent-in-the-loop) 출시입니다: HF는 논문 읽기, 인용 그래프 따르기, 데이터셋 수집/재포맷, 학습 작업 실행, 실행 평가, 실패 반복 등 학습 후 연구 루프를 자동화하는 오픈소스 에이전트인 ml-intern을 발표했습니다 (발표, @lewtun의 지원 게시물, Clement의 프레이밍). 보고된 예시들은 단순히 코딩 데모가 아니라 엔드투엔드 루프이기 때문에 주목할 만합니다: Qwen3-1.7B에서 GPQA 과학적 추론이 10시간 이내에 10% → 32%로 향상되었고, 헬스케어 설정은 HealthBench에서 Codex를 60% 능가했으며, 수학 설정은 완전한 GRPO 스크립트를 작성하고 어블레이션(ablations)을 통해 보상 붕괴에서 회복했습니다. 커뮤니티 테스트는 ml-intern이 자율적으로 파인튜닝하고 아티팩트를 Hub에 다시 게시할 수 있음을 빠르게 보여주었습니다 (SAM 파인튜닝 예시 실행).
- Hermes는 더 풍부한 로컬/오픈 에이전트 플랫폼으로 진화하고 있습니다: 여러 트윗은 Hermes의 실용적인 오픈 에이전트 스택으로서의 모멘텀을 지적합니다: Hermes 에이전트 자체에 의해 생성된 초보자 가이드, Skillkit의 네이티브 지원, Scarf라고 불리는 새로운 macOS GUI, 그리고 로컬 워크플로우에서의 사용 확대입니다. 가장 기술적으로 의미 있는 업데이트는 @Teknium으로부터 나왔습니다: Hermes 서브 에이전트는 이제 더 넓은 스폰 너비와 재귀적 스폰 깊이를 모두 지원하여, 더 깊은 계층적 분해를 가능하게 합니다. 이는 "단일 채팅 루프" 에이전트에서 메모리, 도구, 권한, 재사용 가능한 스킬을 갖춘 다중 프로세스 오케스트레이션 시스템으로의 광범위한 전환과 일치합니다.
- 하네스는 일류 엔지니어링 아티팩트가 되고 있습니다: 트윗 전반에 걸친 반복되는 주제는 에이전트 시스템의 유용한 부분이 기본 모델 단독이 아니라 런타임/하네스라는 점입니다. DSPy 3.2는 RLM 개선 사항과 옵티마이저 체이닝 및 LiteLLM 디커플링을 출시했습니다 (릴리스); Isaac Flath는 RLM이 노트북을 REPL-네이티브 트레이스/평가 인터페이스로서 다시 관련성 있게 만든다고 주장했습니다 (트윗); LangChain은 deepagents 배포를 위한 커스텀 인증을 추가했습니다 (업데이트); 그리고 Claude Code에 대한 논문 요약 스레드는 시스템의 대부분이 순수한 "지능"보다는 하네스 로직임을 강조했습니다 (요약).
Kimi K2.6, KDA 커널, 그리고 시스템적으로 더욱 신뢰할 수 있게 되는 오픈 웨이트 코딩 모델
- Moonshot은 모델 기능과 커널 인프라를 모두 발전시켰습니다: 플래그십 Kimi 스레드는 K2.6이 지속적인 자율성을 가지고 장기 코딩 작업을 완료했다고 주장합니다: 한 번의 실행으로 4,000개 이상의 도구 호출과 12시간 이상에 걸쳐 Zig에서 Qwen3.5-0.8B 인퍼런스를 다운로드하고 최적화하여, 처리량을 약 15 tok/s에서 약 193 tok/s로 개선했으며, LM Studio보다 약 20% 더 빠르게 종료되었습니다 (스레드). 또 다른 실행은 1,000개 이상의 도구 호출과 4,000개 이상의 LOC 변경을 통해 거래 엔진을 재작업하여, 185%의 중간 처리량 및 133%의 최고 처리량 향상을 달성했다고 보고되었습니다 (두 번째 스레드). 이것들은 여전히 벤더 데모이지만, 벤치마크 스크린샷보다는 시스템 작업에 훨씬 더 가깝습니다.
- Kimi는 또한 성능에 중요한 인프라를 오픈소스화했습니다: Moonshot은 FlashKDA를 출시했는데, 이는 CUTLASS 기반의 Kimi Delta Attention 커널 구현으로, H20에서 flash-linear-attention 기준선 대비 1.72배~2.22배의 프리필(prefill) 속도 향상과 flash-linear-attention의 드롭인 백엔드로서의 호환성을 주장합니다 (릴리스). 외부 후속 조치에서는 8x MI300X에서 K2.6 + DFlash가 508 tok/s를 기록하여, 기준선 자기회귀(autoregressive) 설정 대비 5.6배의 처리량 향상을 보였다고 보고했습니다 (HotAisle). DSA/MLA/KDA 변형 모델에 대한 지속적인 논의와 함께, 핵심 신호는 중국 연구소들이 단순히 웨이트를 출시하는 것을 넘어, 실제 배포에 영향을 미치는 어텐션/커널 수준 최적화를 점점 더 많이 발표하고 있다는 점입니다.
- 오픈 웨이트 코딩 품질은 향상되고 있지만, 동등성에 대한 의견 불일치는 여전히 존재합니다: 일부 사용자들은 Kimi K2.6을 최고의 오픈소스/오픈 웨이트 코딩/에이전틱 모델로 간주하는 반면 (@scaling01, Windsurf 가용성), 다른 이들은 프론티어 독점 모델이 WeirdML, 장기 작업, 그리고 신뢰성 면에서 여전히 큰 우위를 점하고 있다고 반박했습니다 (@scaling01 비판, WeirdML 격차). 실질적인 시사점은 "오픈 모델이 따라잡았다"기보다는, 오픈 웨이트 모델이 이제 인프라, 하네스, 배포 품질이 실제 가치의 많은 부분을 결정할 만큼 충분히 신뢰할 수 있게 되었다는 것입니다.
심층 연구 시스템: Google이 연구 에이전트 프론티어를 확장하다
- Google은 Deep Research를 더 구성 가능한 API 프리미티브로 업그레이드했습니다: Google/DeepMind는 Gemini API를 통해 업데이트된 Deep Research와 Deep Research Max를 출시했으며, Gemini 3.1 Pro를 기반으로 협업 계획, 임의의 MCP 지원, 멀티모달 입력(PDF/CSV/이미지/오디오/비디오), 코드 실행, 네이티브 차트/인포그래픽 생성, 실시간 진행 스트리밍 기능을 제공합니다 (Google 스레드, 기능 세부 정보, Sundar 게시물, 개발자 API 게시물).
- 벤치마크 수치는 상업적으로 중요할 만큼 강력합니다: Google은 Max 변형 모델의 DeepSearchQA에서 93.3%, BrowseComp에서 85.9%, HLE에서 54.6%를 기록했다고 강조했습니다 (Sundar, Phil Schmid 요약). 원시 점수보다 더 중요한 것은 워크플로우 디자인입니다: Google은 "야간 실사 / 애널리스트 보고서 생성"을 명확히 제품화하고 있으며, MCP 기반 내부 데이터 접근을 연구 에이전트의 표준 부분으로 만들고 있습니다. 이는 또한 단순한 브라우징 에이전트와 계획, 검색, 코드 실행, 시각 자료 생성, 독점 코퍼스 기반 작업을 수행하는 풀스택 연구 에이전트 간의 격차가 확대되고 있음을 보여줍니다.
리트리벌, 데이터, 그리고 평가: 실제 엔지니어링 가치를 지닌 오픈 릴리스
- 리트리벌 분야에서 LightOn의 의미 있는 오픈 릴리스가 있었습니다: LightOn은 Apache 2.0 라이선스 하에 LateOn과 DenseOn을 출시했는데, 둘 다 149M 파라미터 리트리벌 모델로, LateOn(멀티 벡터/ColBERT 스타일)은 BEIR에서 57.22 NDCG@10을, DenseOn(밀집 단일 벡터)은 56.20을 기록하여, 최대 4배 더 큰 모델들을 능가했습니다 (모델 릴리스, 개요). 그들은 또한 14억 개의 쿼리-문서 쌍을 포함하는 통합 데이터셋 릴리스와 FineWeb-Edu를 기반으로 구축된 새로 고침된 웹 데이터셋을 발표했습니다 (데이터셋 게시물).
- vLLM은 실용적인 배포 지식 레이어를 출시했습니다: recipes.vllm.ai의 재설계는 생각보다 더 유용합니다. 이는 모델 페이지를 실행 가능한 배포 레시피에 매핑하고, 대화형 명령 빌더를 포함하며, NVIDIA 및 AMD를 지원하고, 텐서/전문가/데이터 병렬 변형 모델을 다루며, 에이전트를 위한 JSON API를 노출합니다. 이는 새로운 오픈 모델을 서비스하는 데 있어 운영자 마찰을 줄이는 바로 그런 인프라 문서화 레이어입니다.
- 벤치마크는 단순히 작업 출력뿐만 아니라 에이전트의 사각지대를 점점 더 많이 탐색하고 있습니다: 주목할 만한 예시로는 실제 엔터프라이즈 문서 내 차트 이해를 위한 ParseBench (LlamaIndex, Jerry Liu 세부 정보)와, 해결책이 파일이나 엔드포인트에 문자 그대로 노출되어 있을 때조차 에이전트가 명시적인 환경 단서를 무시하는 경향이 있음을 보여주는 새로운 결과 (논문 스레드)가 있습니다. Google Research의 ReasoningBank 또한 이 주제에 부합하며, 메모리를 성공적인 궤적과 실패한 궤적 모두로부터 학습하는 것으로 구성합니다 (트윗).
